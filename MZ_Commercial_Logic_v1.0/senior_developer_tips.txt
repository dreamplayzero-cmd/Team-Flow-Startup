# Senior Developer's Tips

## Database
- **SQLite Concurrency**: SQLite는 파일 기반이므로 대규모 동시 쓰기에 취약합니다. `WAL(Write-Ahead Logging)` 모드를 활성화하여 동시성 성능을 높이세요.
- **Foreign Keys**: SQLite에서 외래 키 제약 조건은 기본적으로 비활성화되어 있습니다. 연결 시 반드시 `PRAGMA foreign_keys = ON;`을 실행해야 합니다.
- **Indexing**: 자주 필터링되는 컬럼(예: 상권 코드, 지역 코드)에는 인덱스를 설정하여 쿼리 속도를 보장하세요.
- **Parameterized Queries**: `f-string`을 사용한 SQL 생성은 보안의 적입니다. 항상 `?` 플레이스홀더를 사용하세요.

## API Integration (2026-04-17)
- **Decoding Key vs Encoding Key**: `requests` 라이브러리는 파라미터를 자동으로 인코딩(quote)하므로, `.env`에는 **Decoding된 일반 인증키**를 저장하고 사용 시 `unquote()`를 거치는 것이 안전함.
- **API URL 변화**: 공공데이터포털은 주기적으로 엔드포인트를 노후화(Deprecated)하므로 가이드 문서의 `End Point`를 항시 최우선 확인해야 함.
- **날짜 파라미터**: 너무 미래(예: 2026년)를 요청하면 API 서버 내부에서 500 에러를 뱉는 경우가 많으므로, 데이터 존재가 확실한 시점으로 Fallback 처리가 필수.
- **Dual Architecture (Override 전략)**: API가 죽거나 제한에 결렸을 때 깡통(Empty) 데이터 때문에 엔진이 멈추는 것을 막기 위해 1차적으로 50점 부여용 'Fallback/Mock' 로직을 세팅하고, 수동으로 300MB 대용량 정밀 CSV 파일을 삽입(Override)하는 유연한 아키텍처를 권장함.

## Python & Memory (2026-04-17)
- **Pandas Low Memory**: 300MB가 넘는 공공데이터 CSV 파싱 시 `pd.read_csv`에 `low_memory=False` 옵션을 추가하면 자료형 유추 워닝(DtypeWarning) 시스템 부하를 차단할 수 있음.
- **Encoding Exception**: 공공기관 데이터는 종종 `utf-8`과 `cp949` 규칙이 혼재되어 깨질 때가 있음. `FileLoad` 시도 시 `try(utf-8) / except(cp949)` 트랩을 항시 구성할 것.

## UI/UX & Web Integration (2026-04-17)
- **PySide6 QLabel HTML Rendering**: 텍스트에 `<b>` 태그 등을 사용할 땐 반드시 `.setTextFormat(Qt.RichText)`로 설정할 것.
- **Thread Freezing (GUI의 단점 돌파)**: 5초 이상의 데이터 다운로드나 CSV 파싱(`14,000건 insert`) 로직을 UI 스레드에 맡기면 프로그램이 "응답 없음"에 걸리게 됨. 반드시 비즈니스 콜백을 `QThread`나 `Worker` 패턴 구석으로 빼야 함.
- **Decoupling Architecture**: UI 탭(버튼)과 핵심 엔진의 느슨한 결합도를 철저히 유지해야 관리자 대시보드 구조 변경이 용이함.
