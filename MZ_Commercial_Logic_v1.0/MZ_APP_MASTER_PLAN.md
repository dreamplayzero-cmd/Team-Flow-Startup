# 🌟 MZ 상권분석 앱 마스터 플랜 (MZ-Startup Master Plan)

이 문서는 앱의 핵심 비즈니스 로직과 데이터 구조, 개발 로드맵을 정의하는 최상위 가이드라인입니다.

## 🎯 서비스 목표
- **대상**: 소상공인 창업을 꿈꾸는 MZ세대
- **핵심 차별화**: 유동인구 기반이 아닌 **"검색 수요 기반"**의 상권 분석
- **결과물**: 6개 핵심 상권(한남동, 이태원, 성수동, 연남동, 망원동, 가로수길) 점수 및 성공 확률(%) 리포트

---

## 📊 데이터 소스 및 활용 로직

### 1. 주민등록 인구 데이터 (행안부 API)
- **활용**: 타겟 소비자(MZ세대) 거주 밀도 분석

### 2. 상가(상권) 정보 (소상공인 API)
- **활용**: 경쟁 업종 수 파악 (경쟁 역수 계산용)

### 3. SNS 및 검색 트렌드 (네이버 검색 API & 블로그)
- **키워드 생성 로직**: `지역 * 업종 * 니즈` (총 720개 조합 중 유효 키워드 선별)
  - **지역**: 한남동, 이태원, 성수동, 연남동, 망원동, 가로수길
  - **업종**: 카페, 디저트, 브런치, 파스타, 고기집, 술집, 와인바, 치킨, 분식, 베이커리, 버거, 샐러드
  - **니즈**: 맛집, 핫플, 데이트, 분위기, 가성비, 조용한, 혼밥, 추천, 신상, 줄서기
- **검색량 활용**: 최근 검색량(60%) + 증가율(40%) 가중치 적용

---

## 🧠 핵심 분석 알고리즘 (Search-Demand Scoring)

### Step 1. 키워드 점수 산출
> `키워드 점수 = 최근 검색량 * 0.6 + 증가율 * 0.4`

### Step 2. 업종별 수요 점수
> `수요 점수 = (해당 업종 키워드 평균 검색량) / (해당 업종 상가 수)`

### Step 3. 최종 상권 점수 (결정 로직)
> **`상권 점수 = (수요 점수 * 0.4) + (트렌드 증가율 * 0.3) + (경쟁 역수 * 0.3)`**
- *경쟁 역수*: `1 / (경쟁 업체 수 + 1)`

## 🏗 시스템 아키텍처

### [Phase 1] Data Collector (수집기)
- `pop_collector.py`: 인구 데이터 (완료)
- `store_collector.py`: 상기 정보 (완료)
- `blog_crawler.py`: 블로그 크롤링 (완료)
- `naver_collector.py`: 네이버 검색 트렌드 (API 연동 중)

### [Phase 2] Scoring Engine (분석 엔진)
- 상기 **Search-Demand Scoring** 공식을 코드로 구현 (`scoring_engine.py`)
- **[업데이트] 동적 가중치 알고리즘 (Dynamic Weights)**
  - 매장 형태(홀/배달/테이크아웃)에 따라 검색/유동인구/경쟁 가중치 차등 적용
  - 경력(Survival Buffer) 및 자본금(Marketing Buffer)에 따른 베네핏 가산점(+3~+5점) 부여
- 최대 4개 지역 동시 순회 및 최적 상권(1위) 추출 기능 적용 (`main.py`)

### [Phase 3] UI & Report Generation (유저 인터페이스)
- **입력 폼**: 10가지 세분화된 창업자 조건 입력 폼 지원 (`ui/input_view.py`)
- **성공 가능성 문구**: "최적의 상권: OOO, 85%의 성공 가능성" 리포팅
- **장/단점 분석**: 4대 데이터 기준치 돌파 여부에 기반한 다이내믹 텍스트 변환 (`ui/result_view.py`)

### [Phase 4] Automation (수집 자동화) & Dual Architecture
- **Dual Sync & Override 전략**: (1) 기본 실시간 4종 수집 시스템 (2) 수동 대용량 CSV 업로드 덮어쓰기 기능으로 분리.
- `DataSyncWorker`: 4대 데이터 로테이션 수집 시 UI 화면 멈춤(Freezing) 방지를 위한 기본 백그라운드 QThread.
- `CsvImportWorker`: 정밀 상가 분석이 필요할 때 단절 모드(CSV 수동 업로드)로 1.4만 개 상가 데이터를 비동기로 DB에 신속 이식.
- 12시간 주기 스케줄러(QTimer) 작동 지원.

### [Phase 5] User/Admin Separation (권한 계층 분리)
- **LoginView 분기**: 접속 즉시 일반 유저(창업자)와 관리자 모드 분리 진입.
- **AdminDashboard**: 데이터 강제 즉시 동기화(4종) 및 소상공인 상가 고정밀 CSV 다이렉트 업로드 (단절 모드 대응).
- **UserDashboard**: 시각적 방해 요소 없이 창업 자본/연령/조건 분석에만 집중하는 클린한 인터페이스 제공.
